# Start Here

## Use order
1. Open [[00_Dashboard]].
2. Open the current week note in [[02_Program_Index]].
3. Execute the current day's blocks exactly as written.
4. Write outputs into [[50_Learning_Logs]], [[40_Machines]], and [[90_Reports]] as you go.
5. Finish each week in the matching review note under [[03_Weekly_Review_Index]].

## What this vault is for
- It is an execution system for a 24-week offensive cybersecurity program.
- It is not a generic knowledge dump.
- Every note should either drive action, capture evidence, or record a correction.

## Core operating rules
- Stay inside authorized lab-only environments.
- Respect the active and elapsed time budgets.
- Keep Friday as rest.
- Preserve the practical core if you miss a session.
- Update quick references only from real usage.

## Main note types
- Week notes: the exact schedule and deliverables.
- Weekly reviews: what happened, what slipped, what changes next.
- Machine writeups: full target workflow from enumeration through privesc or blocker.
- Learning logs: what changed in your understanding or workflow.
- Report fragments: OSCP-style reporting practice.
