# Week 16 - Exam-style standalone target repetition with stronger Windows and Linux note control

**Phase:** Phase 3 - OSCP Track
**Weekly totals:** 1080 active minutes / 1200 elapsed minutes
**Review note:** [[30_Reviews/Weekly/Week_16_Review]]

## Weekly focus
- increase comfort with PG-style target pacing
- produce report fragments that do not require full note rewrites later
- keep both Linux and Windows workflows alive

## Expected weekly outcomes
- candidate can produce cleaner same-day writeups for standalone targets
- one week's work includes a meaningful report fragment from a fresh target
- enumeration and privesc notes are cleaner under mild pressure

## Saturday - 240 active minutes / 270 elapsed minutes
### Hour 1 - 60m active
**Resource:** Second target or continuation
**Task:** Take a second easy target or push the first target further, but keep notes OSCP-style and concise.
**Tools:** nmap, ffuf or gobuster, browser
**Output:** Second target or continuation note
**Success criteria:** The note remains compact and useful under time pressure.

### Break - 10m
### Hour 2 - 60m active
**Resource:** Exploit validation
**Task:** Pursue the best access path and record only useful attempts and evidence.
**Tools:** Burp Suite, bash, browser
**Output:** Exploit attempt note
**Success criteria:** The note avoids noise and captures decision logic.

### Break - 10m
### Hour 3 - 60m active
**Resource:** Privesc follow-up
**Task:** If a shell is gained, execute the relevant privesc checklist and update the report fragment in parallel.
**Tools:** linpeas or winpeas, bash
**Output:** Privesc and report update note
**Success criteria:** The notes stay synchronized instead of being rewritten later.

### Break - 10m
### Hour 4 - 60m active
**Resource:** Learning log
**Task:** Write a log on how same-day report drafting changed your quality or speed.
**Tools:** Obsidian
**Output:** Learning log: `Week 16 - Same-Day Reporting`
**Success criteria:** The note uses concrete examples from the week's targets.

### Outputs

- Learning log: `Week 16 - Same-Day Reporting`

- The note uses concrete examples from the week's targets.
