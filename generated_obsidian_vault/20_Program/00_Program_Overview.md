# Program Overview

## Goal
Move from Network+ / Security+ foundations into OSCP-track offensive readiness over 24 weeks using only lab-only, authorized environments.

## Weekly cadence
- Saturday: 240 active minutes / 270 elapsed minutes
- Sunday: 240 active minutes / 270 elapsed minutes
- Monday: 120 active minutes / 130 elapsed minutes
- Tuesday: 120 active minutes / 130 elapsed minutes
- Wednesday: 240 active minutes / 270 elapsed minutes
- Thursday: 120 active minutes / 130 elapsed minutes
- Friday: Rest
- Weekly total: 1080 active minutes / 1200 elapsed minutes

## Program design rules
- Breaks are explicit: one 10-minute pause between completed active hours.
- Friday remains rest throughout the program.
- Every week includes at least one learning log and one weekly review.
- Weeks 1-2 include hands-on practical labs.
- Weeks 3-24 include at least one exploit attempt.
- Weeks 4-24 include at least one machine or target writeup.
- Consolidation windows land around Weeks 6, 12, and 18.
- The late phase includes a report-heavy weekend and timed mock work.

## Phase map
- Weeks 1-4: Phase 0 - Foundations - Build networking, Linux, HTTP, note-taking, and first safe-lab workflows.
- Weeks 5-8: Phase 1 - Offensive Entry - Build repeatable enumeration, first exploit attempts, and first real machine writeups.
- Weeks 9-14: Phase 2 - Intermediate Exploitation - Increase independence, target volume, and Linux/Windows privesc routine.
- Weeks 15-24: Phase 3 - OSCP Track - Shift to PEN-200 and Proving Grounds style work, reporting, and timed simulations.

## Tool progression
- Foundations: nmap, Burp Suite, browser dev tools, Linux shell, basic note hygiene.
- Offensive Entry: nmap, Burp Suite, gobuster or ffuf, curl, first machine writeups.
- Intermediate Exploitation: repeated nmap/Burp/content discovery plus linpeas, winpeas, and stronger privesc checklists.
- OSCP Track: PEN-200 and PG workflow with netcat, same-day reporting, timed notes, and mock simulations.

## Output discipline
- Learning logs capture concept-to-practice translation.
- Exploit attempt notes capture payload logic, evidence, and blockers.
- Machine writeups capture enumeration, access, privesc, and lessons learned.
- Weekly reviews keep the schedule realistic and preserve the practical core when sessions slip.

## Safe scope
- All offensive practice is lab-only and limited to THM, HTB, HTB Academy, PortSwigger, OffSec PEN-200, Proving Grounds, or local intentionally vulnerable targets.
