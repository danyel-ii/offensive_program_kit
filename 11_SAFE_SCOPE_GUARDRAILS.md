# Safe-Scope Guardrails

This project is for training and certification preparation.

## Allowed environments
All offensive practice must stay inside:
- TryHackMe
- Hack The Box / HTB Academy
- PortSwigger Web Security Academy
- OffSec PEN-200 / Proving Grounds
- local intentionally vulnerable VMs / containers

## Not allowed
The generated program must not direct the user to:
- attack real public targets
- scan arbitrary internet hosts
- phish real people
- deploy malware
- establish persistence on non-lab systems
- use stealth / evasion techniques outside lab simulations
- steal credentials from real services
- target employers, schools, or public infrastructure

## Framing rule
When a task uses offensive tools, it must be framed as:
- lab-only
- authorized
- contained

## Command rule
Commands may be included only when clearly scoped to:
- a local lab VM / box
- a named training platform room / academy
- a sanctioned practice target

Do not include commands aimed at arbitrary live internet targets.

## Practical target rule
If Codex is uncertain whether a named target still exists, it should:
- choose a stable training room or module instead, or
- describe a current target by platform + difficulty + topic without suggesting real-target alternatives

## Tone rule
The program should cultivate:
- methodical enumeration
- disciplined note-taking
- lawful practice
- exam-style operational thinking in safe environments

It should not cultivate:
- opportunistic real-world targeting
- destructive experimentation
- bypassing authorization
